# 双人时光盒设计文档

日期：2026年6月8日

## 1. 产品定位

双人时光盒是一款面向情侣或亲密伴侣的私密生活瞬间相册。它不做公开社交、不做情侣游戏、不做复杂聊天，而是帮助两个人低负担地共同记录生活片段，并在未来温柔地回看。

产品第一版采用“HarmonyOS 原生完整端 + iPhone Web/PWA 轻量端 + Supabase 后端”的形态。HarmonyOS 端作为主体验，承担完整记录、浏览、相册、纪念日和桌面卡片能力；iPhone 端先用 PWA 覆盖对象的参与需求，包括上传、查看、补一句话和轻回应。

## 2. 目标用户

第一版目标用户是两个人共同维护一段关系记忆的情侣或亲密伴侣，其中至少一方使用 HarmonyOS 设备，另一方可以使用 iPhone。

典型场景：

- 一方拍下日常照片，补一句话，保存为共同回忆。
- 对方在 iPhone PWA 里查看、补一句话或轻回应。
- 双方按时间线回看生活片段。
- 重要纪念日、某年今日、最近合照通过 HarmonyOS 桌面卡片轻量呈现。

## 3. MVP 范围

第一版只做一件事：让两个人共同记录和回看生活瞬间。

包含能力：

- 账号登录。
- 创建双人空间。
- 邀请码或二维码加入双人空间。
- 记录瞬间：照片、文字、日期、心情、地点。
- 共同时间线。
- 共同相册聚合。
- 纪念日管理。
- 某年今日。
- HarmonyOS 桌面卡片。
- iPhone PWA 轻量参与。
- Supabase 数据库、认证和对象存储。

第一版不包含：

- 公开社交。
- 陌生人关系。
- 情侣小游戏。
- 复杂聊天。
- 排行榜、亲密度分数、连续打卡压力机制。
- 大型视频剪辑。
- AI 自动生成长文。
- 端侧加密。
- iOS 原生 App。
- PWA 离线缓存。

## 4. 核心体验

主流程：

```text
登录 -> 创建或加入双人空间 -> 记录瞬间 -> 同步到共同时间线 -> 对方查看并补一句话或回应 -> 未来通过时间线、相册、某年今日回看
```

记录瞬间流程：

```text
点击 + -> 拍照或选择照片 -> 写一句话 -> 选择心情 -> 选择地点 -> 保存
```

保存后支持两种轻互动：

- 对方补一句话。
- 对方选择一个轻回应，例如喜欢、抱抱、想你。

第一版不做评论楼层。每条瞬间最多承载双方各一句话和轻回应，避免产品变成聊天或社交流。

## 5. 页面结构

HarmonyOS 端底部导航使用四个入口：

```text
今天 | 时间线 | 相册 | 我们
```

### 今天

首页承担“打开就看到我们最近生活状态”的作用。

内容：

- 顶部展示在一起第 N 天。
- 主体展示最近一张共同瞬间。
- 展示某年今日。
- 展示下一个纪念日或倒数日。
- 提供快捷记录入口。

### 时间线

按日期分组展示所有瞬间。

每条瞬间包含：

- 照片。
- 双方文字。
- 心情。
- 地点。
- 记录者。
- 发生日期。
- 轻回应。

筛选项：

- 我记录的。
- 对方记录的。
- 收藏的。
- 有地点的。

### 相册

相册不是传统文件夹，而是自动聚合回忆集合。

聚合维度：

- 月份，例如 2026 年 6 月。
- 地点，例如常去的餐厅、学校、旅行城市。
- 纪念日，例如第一次旅行、生日、第一次见面。

### 我们

承载关系空间和设置。

内容：

- 双人空间状态。
- 邀请码和二维码。
- 纪念日管理。
- 同步状态。
- 隐私设置。
- 第一版不展示数据导出入口。

## 6. iPhone PWA 功能

iPhone PWA 保持轻量，只覆盖对象参与共同记录的高频功能。

页面：

```text
记录 | 时间线 | 我们
```

包含能力：

- 登录。
- 通过邀请码或链接加入双人空间。
- 上传照片。
- 写一句话。
- 查看时间线。
- 补一句话。
- 轻回应。
- 查看纪念日。

不包含：

- 离线缓存。
- 复杂相册管理。
- 数据导出。
- 桌面小组件。
- 高级编辑。

## 7. 后端设计

第一版使用 Supabase，原因是它同时覆盖认证、PostgreSQL 数据库、对象存储和 Web/PWA 访问，适合快速跑通跨端同步。HarmonyOS 端通过 REST API 或 Supabase 提供的 HTTP 接口访问后端。

核心数据对象：

```text
User
- id
- displayName
- avatarUrl
- provider
- createdAt

CoupleSpace
- id
- memberIds[2]
- inviteCode
- status
- createdAt

Moment
- id
- coupleSpaceId
- creatorId
- mediaUrls
- text
- partnerText
- mood
- response
- locationName
- occurredAt
- createdAt
- updatedAt
- deletedForUserIds
- isFavorite

Anniversary
- id
- coupleSpaceId
- title
- date
- repeatType
- reminderEnabled
- coverMediaUrl
```

### 权限规则

- 所有业务数据都必须带 `coupleSpaceId`。
- 用户只能读取自己所在双人空间的数据。
- 用户只能创建属于自己双人空间的瞬间。
- 一个双人空间第一版只允许两个成员。
- 图片、视频等媒体上传到 Supabase Storage，数据库只保存 URL 和元信息。
- 删除第一版采用“对自己隐藏”，通过 `deletedForUserIds` 记录，不直接物理删除共同记忆。

### 同步规则

- HarmonyOS 端支持本地缓存。
- HarmonyOS 端支持离线草稿，联网后上传。
- PWA 第一版在线使用，不处理复杂离线冲突。
- `updatedAt` 用于增量同步。
- 同一字段冲突采用最后写入优先。
- 未来若需要更细粒度同步，再拆分双方文字和回应为独立记录。

## 8. HarmonyOS 能力

HarmonyOS 端作为完整主端，使用 ArkTS 和 ArkUI 实现。

核心能力：

- 原生页面和底部导航。
- 相机或相册选择。
- 本地缓存。
- 网络同步。
- 桌面卡片。
- 通知提醒。
- 日期、倒数日和某年今日计算。

桌面卡片内容：

- 最近一条共同回忆。
- 某年今日。
- 下一个纪念日倒数。

第一版不依赖融合开发引擎。该应用是生活类应用，融合开发引擎的 Linux Agent 对核心体验没有必要性。

## 9. 视觉风格

视觉方向是安静、私密、柔和，像一本只属于两个人的生活相册。

原则：

- 照片优先。
- 文字克制。
- 首页不堆功能。
- 配色以暖白、浅灰为主，低饱和粉、绿、蓝作点缀。
- 动效轻量，例如照片淡入、时间线切换、纪念日数字变化。
- 不使用排行榜、连续打卡、亲密度分数等压力机制。

## 10. 测试策略

第一版需要覆盖以下风险点：

- 双人空间权限：非成员不能读取或写入数据。
- 邀请码加入：同一空间最多两个人。
- 媒体上传：上传成功后数据库 URL 可访问。
- 时间线同步：两端新增后都能看到。
- 删除隐藏：只对当前用户隐藏，不影响对方。
- HarmonyOS 离线草稿：断网保存，联网后上传。
- PWA 上传：iPhone Safari 中可以完成登录、上传和查看。
- 桌面卡片：数据为空、有最近回忆、有纪念日三种状态正常。

## 11. 后续扩展

第二阶段可以考虑：

- iOS 原生 App。
- 端侧加密。
- 年度纪念册导出。
- 长图或 PDF 回忆册。
- 地图回忆。
- 更完整的视频支持。
- 更细粒度的共同编辑和撤回。
- 可选 AI：根据照片和文字生成月度回忆摘要，但必须保持私密和可关闭。

## 12. 已确认决策

- 产品方向：温柔相册型情侣共同记忆应用。
- 主端：HarmonyOS 原生 App。
- 对方端：iPhone Web/PWA 轻量端。
- 后端：Supabase 优先。
- MVP 不做 iOS 原生 App。
- MVP 不做社交、游戏、复杂聊天。
- MVP 不使用融合开发引擎作为底座。
